# 💻 Tech Stack & Project Architecture

## 📌 สถาปัตยกรรมระบบ (Frontend & Backend)
จากโครงสร้างโปรเจกต์ที่เราใช้อยู่ สถาปัตยกรรมจะแบ่งเป็น 2 ส่วนหลักๆ ดังนี้ครับ:

### 1. 🖥️ Frontend (ส่วนหน้าบ้าน)
โปรเจกต์นี้ทั้งหมดที่คุณเห็นในโฟลเดอร์ `frontend/` คือส่วนหน้าบ้าน ที่จัดการด้านหน้าตาและการโต้ตอบกับผู้ใช้ 
- **Core Framework:** Next.js 16.2.4 (App Router)
- **UI Library:** React 19.2.4
- **Language:** TypeScript 5.x
- **Styling:** Tailwind CSS v4 + PostCSS

### 2. 🗄️ Backend (ส่วนหลังบ้าน)
ปัจจุบัน **ไม่ได้มีการสร้างโฟลเดอร์ `backend/` สำหรับเขียน Server แยกเอง (เช่น Express, Python)** แต่เราเลือกใช้รูปแบบ Backend-as-a-Service (BaaS) แทน ซึ่งได้แก่
- **Firebase v12.12.1:** รับหน้าที่เป็นหลังบ้านทั้งหมดในตอนนี้ โดยคอยจัดการเรื่อง **ระบบยืนยันตัวตน (Authentication ด้วย Google)** และมีศักยภาพในการเป็นฐานข้อมูล (Firestore) ได้ในอนาคต
- **Next.js Server-Side:** ตัว Next.js เองก็มีคุณสมบัติที่ทำหน้าที่เสมือนกึ่งหลังบ้านได้ (เช่น การประมวลผลก่อนส่งหน้าเว็บ SSR / หรือการทำ API Routes ได้ภายในตัวมันเองโดยไม่ต้องพึ่ง Server อื่น)

## 📁 โครงสร้างโปรเจกต์และหน้าที่ของแต่ละไฟล์

### 1. ⚙️ Root Configuration (ไฟล์ตั้งค่าโปรเจกต์หน้าบ้าน)
นี่คือไฟล์ที่ปรากฏอยู่ชั้นนอกสุดของโฟลเดอร์ `frontend/` ใช้ควบคุมการทำงาน แหล่งรวมการตั้งค่า และไลบรารีต่าง ๆ ของทั้งโปรเจกต์:
- **`package.json`**: ไฟล์เก็บรายชื่อไลบรารีที่โปรเจกต์ใช้งาน (Dependencies) รวมถึงคลังสคริปต์ที่ใช้สั่งงาน เช่น `npm run dev`
- **`package-lock.json`**: เก็บเวอร์ชันของไลบรารีและแพ็กเกจลูกโซ่ทั้งหมดอย่างละเอียด เพื่อป้องกันปัญหา Dependency ขัดแย้งกัน
- **`tsconfig.json`**: ไฟล์คอนฟิกของ TypeScript ควบคุมกฎเกณฑ์ที่เกี่ยวข้องกับการเขียนโค้ดและการตรวจสอบ Types
- **`next.config.ts`**: เครื่องมือตั้งค่าพฤติกรรมทำงานของ Next.js (ใช้ควบคุมพวกข้อจำกัดของฟีเจอร์ต่างๆ)
- **`eslint.config.mjs`**: ตัวคุมกฎระเบียบ (Linter) ในการเขียนโค้ด เพื่อบังคับให้โค้ดของทุกคนในทีมออกมาในมาตรฐานเดียวกัน
- **`postcss.config.mjs`**: ปลั๊กอินทำงานคู่กับ TailwindCSS สำหรับประมวลผลหน้าตาของ CSS
- **`next-env.d.ts`**: เป็นไฟล์ที่ Next.js สร้างโดยอัตโนมัติ เพื่อให้ TypeScript รู้จักประเภทตัวแปรเบื้องหลังของ Next
- **`README.md`**: คู่มือและเกร็ดเล็กเกร็ดน้อยในการเข้าถึงและรันโปรเจกต์สำหรับนักพัฒนา
- **`.gitignore`**: ไฟล์กำหนดว่าไม่ต้องนำไฟล์ไหนบ้างอัปโหลดเข้า Github หรือ Source Control (เช่นโฟลเดอร์ `node_modules` ที่ไฟล์หนักมากๆ)
- **`AGENTS.md`**: ไฟล์เอกสารกำกับสำหรับช่วยให้ AI Agents ฝั่งนักพัฒนา เข้าใจข้อจำกัด โครงสร้างงาน และเป้าหมาย 
- **`CLAUDE.md`**: ไฟล์ตั้งค่าหรือข้อตกลงพิเศษในการทำงานกับ AI (เช่น กฎของการเขียน Prompt / กฎของโปรเจกต์ ที่ Claude หรือ AI ตัวอื่นๆ สังเกตไว้)

### 2. 📂 `src/app/` (ระบบโครงสร้างเนื้อหาและ Routing)
ใช้ App Router ยุคใหม่ของ Next.js ตัวกำหนดโครงสร้าง URL และส่งหน้าเว็บให้ผู้ใช้งาน
- **`layout.tsx`**: โครงหน้าหลักของเว็บทั้งหมด (Root Layout) เปรียบเหมือนโครงกระดูกที่ครอบทั้งหน้า ในที่นี้มีหน้าที่ในการฝัง `AuthContext` ให้ครอบคลุมทุกหน้า
- **`page.tsx`**: คือหน้าแรกของเว็บไซต์ (Landing Page) 
- **`globals.css`**: คลังเก็บ CSS ของทั้งแอปพลิเคชัน เป็นที่ตั้งค่าสี Font และโหลด Tailwind
- **`favicon.ico`**: ไอคอนโลโก้ที่แสดงอยู่บริเวณแท็บด้านบนของเว็บบราวเซอร์ 
- **`year/[number]/page.tsx`**: หน้าโชว์รายวิชาแบบ Dynamic แยกรหัสปีการศึกษา (`/year/1`, `/year/2`) เป็นต้น

### 3. 📂 `src/components/` (ส่วนประกอบ UI ที่ใช้งานซ้ำๆ)
- **`AuthNavbar.tsx`**: ส่วนของแถบ Navigator ด้านบน ทำหน้าที่เป็นทั้งเมนูหลัก และปุ่มกดให้เรียก Login & Logout 
- **`CourseRow.tsx`**: กล่องหน้าตาแสดงผลข้อมูลวิชาต่างๆ (Card) สำหรับให้หน้ารายชั้นปีเอาไปเรียงแสดงเป็น List ยาวๆ

### 4. 📂 `src/contexts/` (ระบบบริหารจัดการสถานะกลาง)
- **`AuthContext.tsx`**: ตัวกระจายข้อมูลสถานะการ Login ไปให้ระบบรู้ เพื่อเช็คว่า "ผู้ใช้มีสิทธิ์ไหม?" และเป็นที่ฝัง Modal หน้าต่าง Popup กั้นให้ Login ด้วย

### 5. 📂 `src/data/` (ข้อมูลดิบออฟไลน์ / Mock Data)
- **`courses.json`**: เป็นฐานข้อมูลจำลอง (Static JSON) สำรองไว้ดึงมาแสดงรายวิชาแทนการยิงเบิกจากฐานข้อมูลฝั่ง Backend หรือ Database ในตอนเริ่มต้น

### 6. 📂 `src/lib/` (โค้ดควบคุมเครื่องมือภายนอก)
- **`firebase.ts`**: รวบรวม Config Keys ต่างๆ เพื่อสั่งงานและคุยกับเซิร์ฟเวอร์ Authentication ของ Google Firebase ให้ใช้งานระบบล็อกอินได้
